# Wallet Cohort & Accumulation Framework

Copy-trading single wallets risks enormous bias if individuals switch models violently. The Cohort Quality layer isolates groupings based upon behavioral properties guaranteeing multiple independent sources fire simultaneously safely.

### 1. Cohort Concentration Limits
A cohort exhibiting a `cohort_concentration_ratio` nearing 1.0 implies all trading capacity flows through precisely connected sub-wallets owned mathematically by exactly one individual. The Quality score natively drops dependencies severely if concentration ratios extend $> 0.55$, triggering native algorithmic warnings preventing tracking individual bagholders.

### 2. Multipliers
Clean distributed cohorts explicitly receive `family_qualifier` execution multipliers enabling safe groups to scale dependencies upward accurately while holding standard mathematical bounds securely.

### 3. Holder Churn Rate
Evaluating `ReturningRatio_24h` helps identify sticky assets from short-term flippable noise sets.
- **Sticky Holders (Returning > 65%)**: Signalizes structural demand, rewarding candidate strategies securely.
- **High Churn (Returning < 40%)**: Indication of massive wallet rotation causing a warning or soft blocker `high_holder_churn` to trigger inside the execution gate limiting dump risks.
