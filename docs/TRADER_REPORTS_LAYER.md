# Trader Reports Engine

A critical requirement separating execution layers natively from blackboxes. Whenever operations execute implicitly offline or via shadow limits, operators need fully transparent visibility tracking why constraints pass explicitly matching mathematical targets cleanly.

## Execution Flows
The engine inherently maps:
- `CandidateMetrics` -> `TraderReportGenerator` -> **Markdown Annotations**.
- Sorts boundaries evaluating `execution_grade` properties cleanly mapping limits safely validating bounds naturally prior to displaying heuristics.

### Graceful Degradation
If metrics are entirely missing, the Report Engine completely bypasses optimistic traces natively printing explicit `~~DEGRADED~~` / `MISSING` boundaries explicitly preventing traders misjudging signal executions confidently mimicking pure reality over assumptions explicitly matching intended boundaries natively.

### Context Options
Auxiliary modifiers (like **Drift Perp Context**) are rendered as `context_only`/`heuristic` checking exactly how outputs limit dependencies intuitively avoiding overriding execution-grade metrics directly.
